핵심 인사이트

문제의 뿌리는 데이터가 없는 게 아니라,
관계값이 없는 것이다.
현재 상황
"데이터는 있다. 연결이 없다."
병원 기록, 처방전, 가족 메시지, 보호사 메모 — 모든 데이터가 존재한다. 그러나 서로 관계를 맺지 못한다. 부모의 혈압 기록이 어제의 식사, 수면과 연결되지 않는다.
→ 맥락 없는 데이터 = 정보 아님
언더더월드 관점
"관계값을 자동 생성해야 한다."
팔란티어의 2중 레이어는 기업 데이터를 연결한다. 언더더월드의 3중 레이어는 사람, 맥락, 모듈을 연결한다. 실시간 맥락 입력 없이 관계값을 자동 생성하는 것이 핵심 차별점이다.
→ 3중 레이어 온톨로지 적용
기회
"가장 개인적인 온톨로지."
부모 돌봄은 극도로 개인적이고 복잡한 맥락이다. AI 에이전트가 100% 맥락을 이해해야만 진짜 도움이 된다. 환각 없는 AI 친화적 데이터 구조가 요구된다.
→ 초개인화 에이전트의 최적 적용처
솔루션 개요
부모님의
디지털 트윈

부모님 한 분 한 분의 건강, 감정, 일상, 관계를 3중 레이어 온톨로지로 표현한다. 데이터를 입력하면 관계값이 자동 생성된다.

Layer 1 — 객체 레이어
사람, 병원, 약물, 증상, 이벤트
Layer 2 — 관계 레이어
복용-처방-진단-보고-감지 관계값
Layer 3 — 모듈 레이어
알림, 에이전트, UGC, 가족 협업
작동 흐름
01
다원 데이터 수집
병원 EMR 연동, 약국 QR 스캔, 요양보호사 앱 입력, 가족 음성 메모 — 모든 경로를 하나의 그래프 DB로 수렴.
02
자동 관계값 생성
맥락 입력 없이 AI가 혈압 기록 ↔ 식사 패턴 ↔ 수면 ↔ 기분을 연결. 인과 그래프 자동 완성.
03
가족 공유 & 에이전트 알림
형제 간 동일한 맥락 공유. AI 에이전트가 이상 패턴 감지 시 담당 가족에게 역할 기반 알림.
04
전문가 레시피 연결
의사, 간호사, 영양사의 돌봄 레시피를 개인 온톨로지에 맞게 커스터마이징해 자동 적용.

왜 이게 달라야 하는가

기존 실버케어 앱과의 근본적 차이 — 데이터 수집이 아닌, 관계값 생성.

01
🕸️
사일로를 그래프로 전환
분절된 의료·감정·일상 데이터를 하나의 지식 그래프로 연결. 데이터가 쌓일수록 관계값이 정교해진다.
→ 맥락 이해율 100%
02
⚡
실시간 입력 없이 작동
기존 헬스케어 앱의 실패 원인 = 입력 부담. 자동 연동 + 최소 입력으로 관계값 자동 생성. 지속 가능한 데이터 증식.
→ 입력 부담 80% 감소
03
🔬
전문가 레시피 구조화
의사·간호사·영양사의 돌봄 노하우를 레시피로 저장. 개인 온톨로지에 맞게 커스터마이징해 자동 적용. 복사·배포 가능.
→ 전문 지식의 민주화
04
🌐
분산 저장 + 가족 협업
특정 플랫폼에 종속되지 않는 분산 구조. 가족 구성원이 UGC 방식으로 데이터를 증식. 돌봄 집단지성 형성.
→ 가족 사일로 해소

언더더월드의 3중 레이어 온톨로지는 부모 돌봄이라는 극도로 개인적이고 복잡한 맥락을 다루기에 최적의 구조다. 시장 진입 전략도 명확하다.

로드맵
Phase 1
MVP — 가족 돌봄 로그
카카오 연동 + 음성 메모로 온톨로지 노드 자동 생성. 가족 대시보드 베타. 초기 타깃: 40~50대 자녀.
Phase 2
병원 EMR 연동 + AI 에이전트
PHR(개인 건강 기록) API 연동. 이상 패턴 감지 에이전트 출시. 역할 기반 알림 시스템 고도화.
Phase 3
전문가 레시피 마켓플레이스
의사·보호사·영양사가 레시피를 올리고 수익화. UGC 기반 데이터 증식. B2B: 요양원·복지관 대시보드 공급.
Phase 4
UTW 플랫폼 확장
부모 돌봄 온톨로지를 언더더월드 메타버스 디지털 트윈 범용 플랫폼의 킬러 버티컬로 확장.
